## 1. Логическая и физическая структура БД (7 стр.)
Информационно-логическая модель: 7 сущностей + 3 junction-таблицы для M:N (поиск/фильтры/группировка через PROC/FN). Атрибуты/типы/ограничения: docs/attributes.md (9 таблиц, 3NF — FK/junction вместо дубликатов, нет транзитивных зависимостей).
Связи: 1:N (Departments 1--N Teachers via DepartmentId; JournalsConferences 1--N Publications via JournalConferenceId; Projects 1--N LeaderId); M:N (Teachers M--N Publications via PublicationAuthors; Teachers M--N Projects via ProjectParticipants; Publications M--N Projects via PublicationProjects). ER-диаграмма: db/schema/er-diagram-v44.png (сущности/атрибуты/связи).
Нормализация: 3NF (FK для 1:N, junction для M:N; CHECK для Type/Rating, UNIQUE для DOI/Number/ISSNISBN).
Физическая модель: MS SQL Server, LocalDB; скрипты CREATE TABLE/FK/INDEX в db/create-schema.sql (позже); диаграмма БД в SSMS; тестовые данные в db/seed-data.sql (10-15 записей).
